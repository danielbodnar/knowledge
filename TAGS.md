## Tag Guidelines

1. **Use lowercase**: All tags should be lowercase
2. **Use hyphens**: For multi-word tags (e.g., `machine-learning`)
3. **Be specific**: Prefer specific tags over generic ones
4. **3-7 tags**: Optimal range per resource
5. **Mix types**: Include technology, domain, use case, and content type
6. **Update this file**: Add new tags as needed with proper categorization

## Adding New Tags

When adding new tags:

1. Check if a similar tag already exists
2. Add to appropriate section
3. Use consistent naming convention
4. Keep alphabetical order within sections
5. Document why the tag is needed (in PR description)

## Tag Maintenance

Periodically review tags to:
- Merge synonyms
- Fix typos
- Remove unused tags
- Consolidate similar tags
- Update deprecated technology names
